Состояние сразу после установки:
1) Уже есть созданный акк админа admin@example.com : admin123

Создание пользователей:
1) Админ добавляет всех новых юзеров - как студентов так и преподов
2) Есть возможность импорта новых студентов из файла в формате '.xlsx'
3) Заполняет всю инфу о пользователях в соответствующем списке: админ, препод, студент - что уже регулирует их права доступа
4) Админ указывает в логине привязанный к юзеру эмейл, а пароль оставляет не установленным
5) Указание группы студента должно быть из списка
5.1) Есть отдельный список групп

Сброс пароля:
1) При добавление админом новых юзеров им на почту приходит ссылка с приглашением установить пароль
2) Сервер проверяет наличие почты и отправляет на нее ссылку для сброса, которая ведет на страницу сброса пароля
3) Пользователь вводит новый пароль

Загрузка приложения:
1) Если пользователь авторизован, то загружается главная, если нет - страница авторизация

Главная:
1) Просмотр списка студентов, преподов и групп
2) Список студентов по ид, группе + пагинация
3) Сортировка списка студентов
4) Выборка студентов по группе

Права доступа (приблизительно):
1) Админ может КРАД всех
2) Препод - только студентов либо всех
3) Студент - только чтение. Мб, вывод только одногруппников
